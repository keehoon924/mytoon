# grill-me

## 목적
사용자의 머릿속에 있는 기획을 빠짐없이 끌어내 명확한 합의에 도달한다.
대충 알겠다고 넘어가지 않는다. 모호한 부분이 남아 있으면 PRD 단계로 못 간다.

## 핵심 원칙

1. **질문은 한 번에 하나씩**
   - 여러 개를 묶어서 던지지 않는다. 답을 받고 다음 질문.
   - 사용자가 한 번에 답하기 쉬워야 한다.

2. **추천 답변을 항상 포함**
   - 객관식 형태로 2~4개 옵션 제시.
   - 가장 합리적인 후보를 "(추천)" 표시.
   - 사용자가 "다른 안 있어?" 하면 추가 옵션 제시.

3. **코드베이스 탐색으로 답할 수 있으면 직접 탐색**
   - 이미 존재하는 모델/스키마/규칙은 사용자에게 묻지 말고 grep/read 로 확인.
   - "현재 X 가 Y 로 구현돼 있던데, 이대로 갈까요?" 같이 확인만.

4. **합의 확인 후 종료**
   - 더 이상 질문할 게 없다고 판단되면 멈춤.
   - "기획 정리가 끝난 것 같습니다. /to-prd 로 PRD 문서를 만들까요?" 라고 물어본다.

## 질문 영역 체크리스트

- 누가 쓰는가 (사용자/페르소나)
- 어떤 문제를 푸는가
- 핵심 사용 흐름 (golden path)
- 엣지 케이스/실패 시나리오
- 범위 외(out of scope) 사항
- 성공 지표
- 기술 제약 (스택, 외부 의존성)
- 데이터 모델/저장 방식
- 인증·권한 모델
- 성능/규모 가정

## 출력 형식

각 질문은 다음 구조로:

```
### 질문 N. <짧은 제목>
<왜 묻는지 한 줄>

- A) <옵션 1> (추천)
- B) <옵션 2>
- C) <옵션 3>
- D) 기타 (직접 입력)
```
