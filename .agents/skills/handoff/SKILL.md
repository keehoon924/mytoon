# handoff

## 목적
대화 컨텍스트가 비대해졌을 때, 새 세션이 0 부터 다시 묻지 않고 곧장 이어서 작업할 수 있도록 한다.

## 출력
`docs/handoff/<YYYY-MM-DD>-<짧은-주제>.md` 파일 1 개.

## 문서 구조

```markdown
# 인수인계: <주제>

작성일: <YYYY-MM-DD>
이전 세션 작업 범위: <한 줄>

## 1. 현재 상태
- 진행 중 이슈: #<번호> (제목)
- 현재 브랜치: main (직접 커밋 규칙)
- 마지막 커밋: <sha 짧게> <메시지>
- 빌드/테스트 상태: pass / fail (사유)

## 2. 무엇이 완료됐나
- ...
- ...

## 3. 무엇이 남았나 (다음 액션)
- [ ] ...
- [ ] ...

## 4. 핵심 결정 사항
- 결정 1: ... (왜)
- 결정 2: ... (왜)

## 5. 함정·주의사항
- 이번 세션에서 발견한 미묘한 부분
- 외부 시스템 제약
- 임시방편 처리한 곳 (TODO 위치)

## 6. 참조 문서
- PRD: `docs/prd/<file>.md`
- 관련 이슈: #N, #M
- 관련 코드: `src/foo.ts:42`

## 7. 새 세션 시작 시 첫 명령
"<위 1~6 요약을 읽고, 다음 액션부터 시작해줘>"
```

## 작성 규칙

1. **민감정보 제거 필수**
   - API 키, 토큰, 비밀번호, 사내 URL, 개인 식별정보 절대 포함 금지.
   - 코드 스니펫은 핵심만, 비밀 값 자리에는 `<REDACTED>`.

2. **기존 산출물은 경로로 참조**
   - PRD/이슈/코드 전체 본문을 복사하지 않는다.
   - 파일 경로·라인 번호·이슈 번호로 가리킨다.

3. **추측 금지**
   - "아마 이럴 것이다" 는 적지 않는다. 확실한 것만.
   - 모르는 것은 "확인 필요" 로 명시.

4. **분량**
   - 1 페이지(~300 줄) 이내가 목표.
   - 길어지면 절을 더 쪼개거나 참조로 뺀다.

## 작성 후
- 파일 경로 출력 → 사용자에게 확인 요청.
- 승인되면 main 에 커밋·푸시.
