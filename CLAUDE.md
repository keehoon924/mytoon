## 스킬 자동 운영 규칙

아래 6개 스킬을 프로젝트 진행 상황에 맞춰 자동으로 판단하고 제안해라.
직접 실행하지 말고, 반드시 나에게 먼저 물어본 후 승인받고 실행해라.

### 진행 순서 및 전환 타이밍

1. /grill-me → 더 이상 질문할 것이 없다고 판단되면:
   "기획 정리가 끝난 것 같습니다. /to-prd 로 PRD 문서를 만들까요?" 라고 물어봐라.

2. /to-prd → PRD 작성이 완료되면:
   "PRD가 완성됐습니다. /to-issues 로 작업을 쪼갤까요?" 라고 물어봐라.

3. /to-issues → 이슈 분할이 끝나면:
   "이슈 등록이 완료됐습니다. 1번 이슈부터 개발을 시작할까요?" 라고 물어봐라.

4. 개발 중 → 하나의 이슈가 끝나면:
   "N번 이슈 완료했습니다. 확인 후 다음 이슈로 넘어갈까요?" 라고 물어봐라.

5. /handoff → 세션이 길어지거나 컨텍스트가 무거워지면:
   "세션이 길어지고 있습니다. /handoff 로 인수인계 문서를 만들까요?" 라고 제안해라.

6. /improve-codebase-architecture → 이슈 5개 이상 완료되면:
   "이슈가 N개 완료됐습니다. 코드 구조 점검을 한 번 할까요?" 라고 제안해라.

### 핵심 원칙
- 스킬 전환 시점을 스스로 판단하되, 실행 전 반드시 나에게 확인받아라
- 내가 "다음" 이라고 하면 다음 단계로 진행
- 내가 "아직" 이라고 하면 현재 단계 계속 진행
- 이슈는 번호 순서대로 하나씩 작업
