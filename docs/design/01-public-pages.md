# MyToon 화면 상세 설계 — 1덩이: 공개 페이지 (로그아웃)

> 기준: 크림+세이지 팔레트 / 제목 Jua·본문 Pretendard / HeroUI·Lucide / 손그림 일러스트(우리 AI 생성)
> 화면마다: **목적 · 진입/이탈 · 레이아웃 · 컴포넌트 · 인터랙션 · 상태 · 모바일**

대상 화면: ① 랜딩 ② 예시 갤러리 ③ 로그인 ④ 회원가입 ⑤ 비밀번호 찾기 ⑥ 비밀번호 재설정 ⑦ 이메일 인증 ⑧ 이용약관 ⑨ 개인정보처리방침

---

## ① 랜딩 (`/`)

**목적**: 방문자가 3초 안에 "뭘 만드는 곳인지" 이해 + 결과물(샘플) 보여주고 가입 전환.

**진입**: 직접 URL·검색·공유 링크 · **이탈**: 시작하기→`/signup`, 로그인→`/login`, 예시→`/examples`

**레이아웃 (위→아래)**
```
┌─ 헤더(sticky, 반투명 크림) ──────────────────────────┐
│ [🎨 MyToon]      사용법 · 예시 · 기능     로그인 [시작하기]│
├─ 히어로 ────────────────────────────────────────────┤
│  [🎁 가입 즉시 무료 크레딧 100개]                         │
│  한 줄로 만드는            │   ┌───┐ ┌───┐            │
│  감성 인스타툰 (Jua)       │   │컷1│ │컷2│  ← 우리 AI │
│  주제 한 문장이면 AI가…    │   └───┘ └───┘  샘플 4컷  │
│  [ 주제 입력창          ] │   ┌───┐ ┌───┐  (손그림) │
│  [무료로 시작하기] [예시보기]│  │컷3│ │컷4│            │
├─ 사용법(#how): 4단계 카드 ──────────────────────────┤
│  ①주제입력 →②템플릿 →③캐릭터 →④완성                  │
├─ 예시(#examples): 샘플 6~8개 그리드 + [예시 더 보기] ──┤
├─ 기능(#features): 4~6 카드 ─────────────────────────┤
│  캐릭터 일관성 / 풀 편집기 / 말풍선·폰트 / 한번에 내보내기 │
├─ 요금(스크롤 하단 1회): 100·500·1000 + "가입 시 100 무료"│
├─ 최종 CTA 배너(세이지 그라데이션) ───────────────────┤
└─ 푸터: 로고 · 이용약관·개인정보·문의 · MyToon 운영팀 ──┘
```

**컴포넌트**: HeroUI Button(primary/secondary)·Chip·Card / 커스텀 샘플컷 패널 / 라이트박스 모달

**인터랙션**
- 헤더 nav 클릭 → 해당 섹션 스무스 스크롤
- 시작하기·CTA → `/signup` / 로그인 → `/login` / 예시 더 보기 → `/examples`
- 히어로 주제 입력창 입력 후 버튼 → `/signup` (입력값을 가입 후 첫 작품 주제로 이어주기)
- 샘플 컷 클릭 → 확대 라이트박스(좌우 컷 넘김)

**상태**: 정적. 샘플 이미지 로드 전 스켈레톤. (빈/에러 거의 없음)

**모바일**: 히어로 1단(텍스트 → 샘플 컷 가로 캐러셀) / nav 햄버거 / 갤러리 2열. 결과물 보기는 모바일 완벽.

## ② 예시 갤러리 (`/examples`)

**목적**: 우리 AI로 만든 샘플 인스타툰을 충분히 보여줘 "이런 게 나온다" 확신 + 가입 전환.
※ 운영자 큐레이션 **정적 샘플** (유저 작품 공유 아님 — PRD OOS 유지)

**진입**: 랜딩 nav·더보기 · **이탈**: 시작하기→`/signup`

**레이아웃**: 헤더(랜딩 공통) → 타이틀 "이런 인스타툰이 나와요" → 필터 바 → 샘플 그리드 → CTA 배너 → 푸터
- 필터: `아트스타일(전체·감성손그림·심플카툰·수채화)` + `주제(일상·재테크·힐링·직장…)`

**컴포넌트**: 필터 Chip/Tabs · 샘플 카드(썸네일+제목+컷수+스타일 태그) · 라이트박스 모달

**인터랙션**: 필터 클릭→그리드 필터링 / 카드 클릭→전체 컷 라이트박스(좌우 넘김, 닫기) / 시작하기→가입

**상태**: 로딩=스켈레톤 그리드 / 빈(필터결과 0)="해당 조건 샘플이 곧 추가돼요" / 에러=재시도 버튼

**모바일**: 2열 그리드 / 필터 가로 스크롤 / 라이트박스 풀스크린 스와이프

## 공통 — 인증 카드 패턴 (③~⑦)

모든 인증 화면은 **중앙 단일 카드** + (데스크탑) 좌측 마스코트 일러스트 2단 레이아웃. 모바일은 카드만 단일 컬럼.
```
┌───────────────┬───────────────────────┐
│  마스코트       │   [🎨 MyToon]           │
│  일러스트       │   제목(Jua)             │
│ (데스크탑만)    │   [입력 필드…]          │
│                │   [ 주요 버튼 ]         │
│                │   보조 링크들           │
└───────────────┴───────────────────────┘
```

## ③ 로그인 (`/login`)
**목적**: 기존 사용자 로그인.
**진입**: 랜딩 로그인·가입 "이미 계정"·보호페이지 리다이렉트 · **이탈**: 성공→`/dashboard`(또는 직전 페이지), 비번찾기, 회원가입
**필드**: 이메일·비밀번호 / 버튼 "로그인" / 링크 "비밀번호 찾기" · "회원가입"
**인터랙션**: 제출→검증→성공 리다이렉트 / 실패 인라인 에러 "이메일 또는 비밀번호가 올바르지 않아요" / 링크 이동
**상태**: 기본 · 제출중(버튼 "확인 중…") · 에러(인라인) · 성공
**모바일**: 마스코트 숨김, 카드만

## ④ 회원가입 (`/signup`)
**목적**: 신규 가입 + 무료 크레딧 100 지급 + 인증 메일 발송 → 온보딩.
**진입**: 랜딩 시작하기·CTA, 로그인 "회원가입" · **이탈**: 성공→**온보딩 투어**
**필드**: 이메일 · 비밀번호(8자+) · **약관 동의 체크박스**(이용약관·개인정보 링크) / 뱃지 "가입 즉시 무료 100개" / 버튼 "시작하기" / 링크 "이미 계정 있어요? 로그인"
**인터랙션**: 검증(이메일형식·비번8자·약관체크)→성공 시 세션 쿠키+온보딩 / 중복 이메일 에러 / 약관 미동의 시 버튼 비활성 / 약관·개인정보 링크는 새 탭
**상태**: 기본 · 필드별 인라인 검증 에러 · 제출중 · 성공(온보딩 이동) · "인증 메일 보냈어요" 토스트
**비고**: 현재 코드엔 약관 체크박스 없음 → **추가 필요**. 가입 크레딧 50→**100** 변경.
**모바일**: 단일 컬럼

## ⑤ 비밀번호 찾기 (`/forgot-password`)
**목적**: 재설정 메일 발송.
**필드**: 이메일 / 버튼 "재설정 메일 받기" / 링크 "로그인으로"
**인터랙션**: 제출→**보안상 존재여부 노출 없이** "가입된 이메일이면 메일을 보냈어요" 안내
**상태**: 기본 · 제출중 · 완료(안내) · 에러

## ⑥ 비밀번호 재설정 (`/reset-password?token=`)
**목적**: 토큰으로 새 비번 설정.
**필드**: 새 비밀번호 · 비밀번호 확인 / 버튼 "변경하기"
**인터랙션**: 진입 시 토큰 검증 → 유효=폼 / 만료·무효="링크가 만료됐어요" + 재요청 링크. 변경 성공→`/login` + 토스트
**상태**: 토큰검증중 · 유효 · 만료/무효 · 제출중 · 성공

## ⑦ 이메일 인증 (`/verify-email?token=`)
**목적**: 인증 토큰 확인.
**레이아웃**: 진입 시 자동 검증 → 성공 "인증 완료! 🎉" + [내 서랍으로] / 실패 "링크가 만료됐어요" + [인증 메일 재발송]
**상태**: 검증중(스피너) · 성공 · 만료/무효

## ⑧ 이용약관 (`/terms`) · ⑨ 개인정보처리방침 (`/privacy`)

**목적**: 법적 고지 (내가 한국어 초안 작성).
**레이아웃**: 헤더 → 제목 → (목차) → 본문(최대폭 ~720px, Pretendard, 섹션 구분) → 푸터
**내용 근거**: 운영주체 "MyToon 운영팀" / 담당 edcrfv51@gmail.com / 수집=이메일·비번해시·이용기록 / **베타 데이터 삭제·초기화 가능 조항 포함** / 결제정보 제외(베타 무료)
**인터랙션**: 목차 클릭→섹션 스크롤
**상태**: 정적 / **모바일**: 단일 컬럼

### 결정 사항 (확정)
1. ✅ 랜딩 히어로 주제 입력창 → **가입 후 첫 작품 주제로 이어줌** (입력값 보존 → 온보딩/새작품 위저드에 프리필)
2. ✅ 이메일 인증 **전에도 앱 사용 가능**, 상단에 "이메일 인증해주세요" **배너만** 표시
3. ✅ 예시 갤러리 = **랜딩 미리보기(6~8개) + 전용 `/examples` 페이지 둘 다**
