# MyToon 화면 상세 설계 — 3덩이: 어드민 (운영자 전용)

> 기준: 동일 디자인 토큰·컴포넌트(HeroUI·Lucide) 적용하되 **기능 우선·정보 밀도 높은** 레이아웃 (마케팅 화려함 X).
> 접근: `role === ADMIN`만. 일반 사용자 접근 시 403/홈 리다이렉트.
> 어드민은 **운영자(PC) 가정** — 반응형만 보장, 모바일 최적화는 후순위.
> 화면마다: **목적 · 진입/이탈 · 레이아웃 · 인터랙션 · 상태**

대상: ⑰ 통계 대시보드 ⑱ 사용자 관리 ⑲ 작품 관리 ⑳ 신고·피드백 큐 ㉑ 프리셋 캐릭터

**어드민 공통 레이아웃**
```
┌ 좌측 사이드바 ─┬ 메인 ───────────────────────────┐
│ 📊 통계        │                                  │
│ 👥 사용자       │   (각 페이지 콘텐츠)              │
│ 🖼 작품         │                                  │
│ 🚩 신고·피드백  │                                  │
│ 🧑‍🎨 프리셋     │                                  │
│ ─────          │                                  │
│ ← 앱으로        │                                  │
└────────────────┴──────────────────────────────────┘
```

## ⑰ 통계 대시보드 (`/admin`)

**목적**: 핵심 운영 지표 한눈에 (베타 성과 추적).
**진입**: 어드민 첫 화면 · **이탈**: 각 관리 페이지

**레이아웃**
- KPI 카드: **WAU(주간 활성)** · 총 사용자 · 총 작품 수 · 총 컷 생성 수 · 미처리 신고/피드백
- 차트: 일별 가입 추이 · 일별 작품/컷 생성 추이
- 최근 활동: 최근 가입 5 · 최근 작품 5 · 미처리 신고/피드백 바로가기
- (매출은 베타 결제 OFF라 "베타 — 무료 운영" 표기)

**인터랙션**: 기간 필터(7일/30일/전체) · KPI 카드 클릭 → 해당 관리 페이지
**상태**: 로딩=스켈레톤 / 데이터 / 에러

---

## ⑱ 사용자 관리 (`/admin/users`)

**목적**: 사용자 조회·검색·크레딧 수동 지급·역할/상태 관리.
**진입**: 사이드바, 통계 카드 · **이탈**: 사용자 상세

**레이아웃**: 검색바 + 테이블
- 컬럼: 이메일 · 가입일 · 마지막 활동 · 크레딧 잔액 · 역할(USER/ADMIN) · 작품 수 · 상태

**인터랙션**
- 검색(이메일) · 정렬(가입일/활동/크레딧)
- 행 클릭 → 상세 드로어: **크레딧 수동 지급(+N, 사유)** · 역할 변경 · 정지/삭제(확인)
- admin 본인은 크레딧 무제한(차감 안 됨)

**상태**: 로딩 / 빈(검색 결과 없음) / 에러

---

## ⑲ 작품 관리 (`/admin/projects`)

**목적**: 전체 작품 조회·검토·삭제 (신고 대응·운영).
**진입**: 사이드바, 신고 큐에서 대상 작품 · **이탈**: 작성자/신고 항목

**레이아웃**: 검색/필터(작성자·상태·기간) + 카드 그리드(썸네일·제목·작성자·생성일·상태)
**인터랙션**: 작품 클릭 → 미리보기 모달(컷 전체 보기) · 삭제(확인 모달) · 작성자로 이동
**상태**: 로딩=스켈레톤 / 빈 / 에러

---

## ⑳ 신고·피드백 큐 (`/admin/reports`)

**목적**: 사용자 신고 + **베타 피드백**을 한 곳에서 처리 (10.A 통합).
**진입**: 사이드바, 통계 미처리 카드 · **이탈**: 대상 작품/캐릭터

**레이아웃**: 탭/필터 `[신고] [피드백: 버그·요청·문의]` + 목록
- 신고 행: 유형(작품/캐릭터) · 대상 · 사유 · 신고자 · 일시 · 상태
- 피드백 행: 카테고리 · 내용 미리보기 · 보낸 사람 · 일시 · 상태

**인터랙션**
- 항목 클릭 → 상세: 대상 콘텐츠 미리보기
- 신고 처리: **삭제 / 반려 / 보류** (기존 `ReportStatus`: RESOLVED_DELETED · RESOLVED_DISMISSED · RESOLVED_HOLD)
- 피드백 처리: 읽음 · 완료 표시 (선택: 운영 메모)

**상태**: 미처리 배지 카운트 · 로딩 · 빈("처리할 항목이 없어요")

## ㉑ 프리셋 캐릭터 관리 (`/admin/presets`)  ★ "기본 캐릭터" 라이브러리

**목적**: 모든 사용자가 고를 수 있는 **기본(프리셋) 캐릭터**를 운영자가 등록·관리.
(그림 못 그리고 사진도 없는 사용자가 바로 쓰는 캐릭터)
**진입**: 사이드바 · **이탈**: 잔류

**레이아웃**: 프리셋 카드 그리드(썸네일·이름·아트스타일 태그) + [+ 새 프리셋]
**새/수정 프리셋 폼**:
- 이름
- 캐릭터 이미지 (업로드 또는 우리 AI로 생성)
- `descriptionPrompt` (일관성용 외모 설명 — 25.A 참조이미지 기반 생성에 사용)
- 아트스타일 태그(감성손그림/심플카툰/수채화)

**인터랙션**: 추가 · 수정 · 삭제(확인) · 미리보기
**상태**: 로딩 / 빈("등록된 프리셋이 없어요 — 첫 캐릭터를 만들어보세요") / 에러

**비고(25.A 연결)**: 캐릭터 일관성이 참조 이미지 기반이므로, 프리셋은 **좋은 기준 이미지**(정면·깔끔)를 갖는 게 중요. 표정/각도 변형 세트는 베타 후 확장.

### 검토 포인트 (확인 필요)
1. **피드백 저장 구조**: 베타 피드백(대상 없는 일반 메시지)을 (a) 기존 `Report` 모델 확장 / (b) **새 `Feedback` 모델** 추가. 현재안 = (b) 별도 모델(신고와 성격이 달라 깔끔). 어드민 큐에서는 탭으로 같이 봄
2. **어드민 디자인 수준**: 동일 컴포넌트·토큰 쓰되 (a) **기능 우선 심플**(현재안) / (b) 앱과 동일한 감성 톤까지. 어드민은 본인만 보니 (a) 권장
3. **프리셋 초기 세트**: 베타 오픈 시 기본 프리셋 캐릭터를 **몇 개·어떤 캐릭터**로 시작할지 (예: 여성/남성/커플/동물 6~8종 — 샘플 인스타툰과 같은 톤으로 우리 AI 생성)
