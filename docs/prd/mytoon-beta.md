# MyToon 베타 출시 + 전면 리디자인 PRD

> 기존 `docs/prd/mytoon.md`(초기 기획)의 후속. 기능은 대부분 구현됐고, 이번 PRD는
> **실유저 베타 출시**를 목표로 (1) 디자인 전면 통일 (2) 핵심 미완·반쪽 기능 보강
> (3) 운영 준비를 다룬다. 화면별 상세 설계는 `docs/design/01·02·03` 참조.

## 1. 문제 정의

- **누가**: 그림 못 그리는 일반인(20~30대 중심), 인스타에 감성 만화를 올리고 싶은 사람.
- **어떤 상황**: 제품의 기능은 거의 다 만들어졌으나, (a) 화면 디자인이 제각각이라 첫인상이 휑하고, (b) **핵심 셀링포인트인 "캐릭터 일관성"이 실제로는 작동하지 않으며**, (c) 말풍선·한글 폰트 등 인스타툰 외관 디테일이 비어 있고, (d) 약관·모더레이션·분석 등 운영 준비가 안 됨.
- **무엇이 불편한가**: 지금 상태로 사용자를 받으면 "캐릭터가 컷마다 딴 사람", "한글 폰트 깨짐", "그린 인페인팅이 적용 안 됨" 등으로 신뢰를 잃는다.

## 2. 해결 방안

### 한 줄 요약
크림+세이지 손그림 감성으로 21개 화면을 통일하고, 캐릭터 일관성·말풍선·폰트 등 핵심 품질 구멍을 메운 뒤, 무료 베타(~50명)로 출시한다.

### 핵심 아이디어
1. **디자인 시스템 1개로 통일** — 토큰(색·폰트·반경·그림자·컴포넌트)을 정의하고 모든 화면에 적용. 손그림 일러스트가 외관의 핵심.
2. **캐릭터 일관성 재구현** — 텍스트 설명만 넘기던 방식 → 등록한 캐릭터의 **참조 이미지를 매 컷 생성에 사용**(gpt-image-1).
3. **인스타툰 외관 보강** — 진짜 만화 말풍선 셰이프 + 한글 폰트 라이브러리.
4. **무료 베타 운영** — 가입 크레딧 100, 결제 OFF, AI 실패 자동 환불, 피드백 폼, 분석/에러 트래킹.

### 왜 이 방식인가 (대안 검토)
- **디자인**: 페이지별 임시방편 대신 토큰+HeroUI 컴포넌트 라이브러리로 일관성 확보. 어드민까지 동일 톤 적용(운영자 만족도·일관성).
- **일러스트**: 자체 AI 생성 또는 사용자 제공 이미지. 베타에선 **사용자가 GPT로 생성→전달** 방식 채택(품질·비용 0). 실제 이미지 배치는 개발 후반 일괄 진행, 그 전엔 이모지 placeholder.
- **캐릭터 일관성**: LoRA 학습(과함, OOS) 대신 참조 이미지 기반 img2img. gpt-image-1로 생성·편집 통일.
- **결제**: 베타는 무료. 50명 규모에선 결제 검증 통계가 안 나오고, 사용자 확보가 우선.
- **컴포넌트**: HeroUI(React 19·Tailwind 4 호환 확인). 디자인 자유도 + 접근성 무료.

## 3. 사용자 스토리

- **US-1 (방문자 전환)**: 방문자로서, 랜딩에서 "무엇을 만들 수 있는지(예시)"와 "어떻게 쓰는지"를 3초 안에 파악하고 가입할 수 있다.
  - 수용: 히어로 주제 입력창에 적고 시작하면 가입 후 그 주제로 첫 작품이 이어진다.
- **US-2 (가입/인증)**: 사용자로서, 이메일+비밀번호로 가입(약관 동의)하고 무료 크레딧 100개를 받는다.
  - 수용: 이메일 인증 전에도 앱 사용 가능, 상단에 인증 배너만 표시. 인증 링크 클릭 시 성공/만료 상태가 보인다.
- **US-3 (온보딩)**: 신규 가입자로서, 가입 직후 온보딩 투어로 첫 작품 생성까지 안내받는다.
  - 수용: 환영 → 흐름 미리보기 → 새 작품 위저드 핸드오프. 건너뛰기 가능, 1회만 자동.
- **US-4 (캐릭터 등록)**: 사용자로서, 사진/텍스트/프리셋(기본 캐릭터) 중 하나로 캐릭터를 등록해 재사용한다.
  - 수용: 등록 캐릭터가 위저드·편집기에서 선택 가능.
- **US-5 (캐릭터 일관성)**: 사용자로서, 등록한 캐릭터가 모든 컷에 **일관된 모습**으로 등장한다.
  - 수용: 컷 생성 시 캐릭터 참조 이미지가 실제로 사용되어 "같은 캐릭터"로 인식된다.
- **US-6 (작품 생성)**: 사용자로서, `템플릿 → 아트스타일 → 캐릭터 → 주제`를 거쳐 AI 자동/직접 모드로 4~10컷을 생성한다.
  - 수용: 생성 직전 잔액·차감량 표시. 실패한 컷은 크레딧 자동 환불 + 안내.
- **US-7 (템플릿)**: 사용자로서, 레이아웃+구조+아트스타일이 묶인 템플릿을 골라 쉽게 시작한다.
  - 수용: 위저드 첫 단계. 아트스타일 3종(감성손그림/심플카툰/수채화) 별도 선택.
- **US-8 (편집기)**: 사용자로서, 생성물을 말풍선·브러시·필터·레이어로 다듬고, 컷 재생성·부분 인페인팅한다.
  - 수용: 진짜 만화 말풍선 10종+, 한글 폰트 적용, 멀티 캐릭터 추가, 인페인팅 마스크가 실제 반영. 30초 자동저장·버전 복원·undo.
- **US-9 (Export)**: 사용자로서, PNG/JPG/ZIP/클립보드로 내보내 인스타에 올린다. 모바일에서도 완벽.
- **US-10 (설정 통합)**: 사용자로서, 설정 한 곳에서 계정·크레딧·신고/피드백을 관리한다.
  - 수용: 상단 nav는 기능만(내 서랍·캐릭터·사용법). 크레딧 잔액은 프로필 드롭다운+생성 직전 표시.
- **US-11 (피드백)**: 사용자로서, 사이트 내 폼으로 버그/요청/문의를 보낸다 → 어드민 큐로 수집.
- **US-12 (어드민)**: 운영자로서, 통계·사용자·작품·신고/피드백·프리셋 캐릭터를 관리하고 크레딧을 수동 지급한다.
- **US-13 (콘텐츠 안전)**: 시스템으로서, 생성 직전 프롬프트를 모더레이션해 부적절 입력을 차단한다.

## 4. 구현 결정

### 스택 / 라이브러리
- 기존: Next.js 16(App Router)·React 19·TypeScript·Tailwind 4·Prisma 7·PostgreSQL·OpenAI·S3/R2·Toss.
- 추가: **HeroUI**(컴포넌트, React19·TW4 호환 확인), **Lucide**(아이콘), **Pretendard**(본문, CDN)·**Jua**(제목, Google Fonts), **sharp**(인페인팅 마스크 적용), **Sentry**+**Vercel Analytics**(에러·분석).
- 이유: 위 "왜 이 방식인가" 참조.

### 디자인 토큰
- 크림+세이지 9색 + 시맨틱 토큰, 반경, 그림자 4단, 모션 — `src/app/globals.css`에 CSS 변수로 정의(완료). 스타일 가이드 전문은 대화 기록/`docs/design` 참조.

### 데이터 모델 (변경/추가)
- `User`: **`onboardedAt DateTime?`** 추가(온보딩 1회 처리). `lastActiveAt` 기존.
- `Feedback`(신규): `id, userId, category(BUG|REQUEST|INQUIRY), message, status(OPEN|DONE), createdAt`. 신고(`Report`)와 분리, 어드민 큐에서 탭으로 함께 조회.
- `Character`/`PresetCharacter`: `referenceImageUrl`(기존)을 **생성에 실제 사용**. 아트스타일 태그 필요시 추가(TBD).
- `Cut`: 생성 시 캐릭터 `referenceImageUrl` 입력으로 사용(스키마 변경 없음, 로직 변경).
- `Bubble`: `bubbleType` 10종+로 확장(셰이프), `font`는 실제 한글 웹폰트 키로. `color/fontSize`(기존) 유지.
- 가입 크레딧 50→**100**(완료).
- 템플릿: 정적 설정(코드 상수) 또는 신규 테이블(TBD — 수직 슬라이스에서 결정).

### API 엔드포인트 (변경/추가)
- `POST /api/projects/:id/generate`: **참조 이미지 기반 생성**으로 변경(gpt-image-1), 템플릿/아트스타일 반영, **실패 컷 자동 환불**.
- `POST /api/projects/:id/cuts/:cutId/inpaint`: **sharp로 마스크 실제 적용**.
- 생성 직전 **모더레이션**(OpenAI moderation) 호출.
- `POST /api/feedback`, 어드민 `GET /api/admin/feedback`: 신규.
- 인증 메일 **재발송** 엔드포인트(설정에서): 신규.
- `verify-email` API: redirect → **JSON 응답**으로 변경(완료).

### UI 화면 흐름 (21개)
- 공개: 랜딩 → 예시갤러리 → 인증5종 → 약관/개인정보
- 앱: 온보딩 → 내 서랍 → 새 작품 위저드(템플릿→아트스타일→캐릭터→주제) → 프로젝트 뷰 → 편집기 → Export → 설정
- 어드민: 통계 → 사용자 → 작품 → 신고/피드백 → 프리셋
- 각 화면 상세: `docs/design/01-public-pages.md`·`02-app-core.md`·`03-admin.md`

### 인증 · 권한
- 일반 user: 본인 데이터만. 이메일 인증 전에도 사용 가능(배너).
- admin: 전체 데이터·크레딧 무제한·프리셋/신고/피드백 관리.

### 외부 의존성
- OpenAI(gpt-image-1 생성·편집, moderation, GPT-4o 시나리오) — **베타 운영에 키 필수**.
- S3/R2(이미지), Toss(결제 — 베타 OFF), Resend(메일), Sentry, Vercel Analytics.

## 5. 테스트 결정
- **단위**: 크레딧 차감·**실패 시 환불** 원자성, 권한 체크, 모더레이션 차단, 온보딩 1회 처리.
- **통합/E2E**: 가입(약관)→온보딩→템플릿/캐릭터→생성→편집→Export 골든패스. 크레딧 부족 차단. 캐릭터 일관성(참조 이미지 사용 여부). 인페인팅 마스크 반영.
- **수동 QA**: 21개 화면 디자인 일관성, 모바일 깨짐 여부(결과물 보기/공유), 편집기 PC 동작, 어드민 큐(신고+피드백), 프리셋 캐릭터 생성.
- **성공 지표**: WAU(`lastActiveAt`), 작품/컷 생성 수, 가입→첫 작품 도달률(온보딩 효과). 어드민 통계 대시보드 노출. 정성: 베타 피드백 폼 수집.

## 6. 범위 외 (Out of Scope)
- 실제 결제 활성화(베타 무료, 코드는 유지·OFF).
- 모바일 편집기(편집은 PC 권장, 보기·내보내기만 모바일).
- 협업 편집, 유저 작품 공개 갤러리, 소셜 로그인, 인스타 직접 게시, LoRA 캐릭터 학습.
- 아트스타일 4종 이상 확장(베타는 3종).
- 이미지 자동 모더레이션(텍스트 모더레이션만; 이미지 검수는 후순위).
- 변호사 정식 약관 검토(베타는 자체 표준 초안).

## 7. 오픈 이슈
- 템플릿을 정적 상수로 둘지 DB 테이블로 둘지 (수직 슬라이스 분할 시 결정).
- 캐릭터/프리셋에 아트스타일 태그 컬럼 추가 여부.
- 인증 메일 재발송 rate limit 정책.
- 실제 일러스트 파일 전달·배치 시점(개발 후반 일괄) — 전달 방식(GitHub 업로드 등) 미확정.
- HeroUI v3 실제 통합 시 Tailwind4 플러그인 설정 세부.
